# Plan de Implementación y Aprendizaje de EduFlow

¡Bienvenido al desarrollo de **EduFlow**! Contar con un plazo de **2 meses** es una excelente oportunidad para aprender a fondo no solo Django, sino también buenas prácticas de diseño de software, bases de datos, tareas asíncronas con Celery, APIs REST y frontend moderno con HTMX y Tailwind CSS.

Este plan está estructurado para avanzar a un ritmo constante pero detallado, asegurando que entiendas el **porqué** y el **cómo** de cada decisión técnica.

---

## Planificación General (2 Meses)

Dividiremos el proyecto en **6 fases quincenales**:

| Fase | Duración | Foco Principal | Tecnologías a Aprender |
|:---|:---:|:---|:---|
| **Fase 1: Estructura y Entorno** | Semanas 1-2 | Inicializar el proyecto, entorno virtual, PostgreSQL, y arquitectura modular. | Python `venv`, Django 5, PostgreSQL, Git. |
| **Fase 2: Usuarios y Roles** | Semanas 3-4 | Modelos de usuario personalizados, roles (admin, profesor, etc.) y Django Admin. | Django ORM, Custom User Model, Django Admin, Fixtures. |
| **Fase 3: Gestión Académica** | Semanas 5-6 | Cursos, inscripciones, sesiones y control de asistencia interactivo. | Django Templates, Django Managers, HTMX (asistencia en tiempo real). |
| **Fase 4: Pagos y Tareas Asíncronas** | Semanas 7-8 | Registro de pagos, generación de PDFs y envío de correos en segundo plano. | Celery, Redis, WeasyPrint, Django Signals. |
| **Fase 5: API REST y JWT** | Semanas 9-10 | Creación de endpoints y seguridad con tokens para integraciones. | Django REST Framework (DRF), SimpleJWT, Swagger. |
| **Fase 6: Pulido y Despliegue** | Semanas 11-12 | Diseño visual premium, Tailwind CSS y despliegue a producción. | Tailwind CSS, Alpine.js, Docker, Railway. |

---

## Fase 1: Configuración del Entorno e Inicialización del Proyecto (Actual)

En esta primera fase, estableceremos una base sólida y limpia para el proyecto. Evitaremos la estructura por defecto desorganizada y utilizaremos una arquitectura modular recomendada para proyectos profesionales.

### Objetivos de la Fase 1
1. **Configurar el entorno de desarrollo:** Instalar Python 3.12, crear un entorno virtual e instalar las dependencias clave.
2. **Crear el proyecto Django con arquitectura modular:** Separar la configuración central de las aplicaciones de negocio en una carpeta `apps/`.
3. **Configurar la Base de Datos:** Establecer PostgreSQL o configurar una base de datos local para desarrollo rápido (SQLite) mientras configuramos PostgreSQL.
4. **Configurar las variables de entorno:** Mantener seguras las credenciales y claves con `django-environ` o `python-dotenv`.

## Propuesta de Estructura de Archivos

Organizaremos el proyecto de la siguiente manera:

```
PROYECTO_FINAL/
├── .env.example            # Plantilla para variables de entorno
├── .gitignore              # Archivos excluidos de Git
├── requirements.txt        # Dependencias del proyecto
├── manage.py               # Comando de gestión de Django
├── core/                   # Configuración del proyecto principal
│   ├── __init__.py
│   ├── asgi.py
│   ├── celery.py           # Configuración de Celery
│   ├── settings.py         # Ajustes de Django
│   ├── urls.py             # Rutas principales del proyecto
│   └── wsgi.py
├── apps/                   # Contenedor de nuestras aplicaciones de negocio
│   ├── __init__.py
│   ├── accounts/           # Gestión de usuarios, roles y perfiles
│   ├── courses/            # Cursos, inscripciones, sesiones y asistencia
│   └── payments/           # Pagos, facturas e historial de correos
├── static/                 # Archivos estáticos globales (CSS, JS, imágenes)
└── templates/              # Plantillas HTML globales
```

## Pasos para Iniciar Hoy (Fase 1 - Paso A)

Para dar el primer paso de forma segura y controlada, realizaremos las siguientes tareas:

### 1. Creación del archivo de Dependencias (`requirements.txt`)
Definiremos las librerías necesarias para iniciar, incluyendo:
- Django 5.0
- Django REST Framework 3.15
- `django-environ` para variables de entorno
- `psycopg2-binary` para la conexión con PostgreSQL

### 2. Creación del Entorno Virtual e Instalación de Dependencias
Te guiaré paso a paso para crear tu entorno virtual `venv` y activar el intérprete adecuado en tu terminal de Windows.

### 3. Creación del Proyecto Django
Inicializaremos el proyecto con la estructura de carpeta personalizada `core` para evitar la duplicidad de nombres típica de Django.

---

## Plan de Verificación

Para asegurar que todo funcione correctamente antes de avanzar:
- **Verificación de dependencias:** Ejecutaremos comandos para confirmar que las librerías se instalaron en el entorno virtual correcto.
- **Verificación del servidor de desarrollo:** Iniciaremos `python manage.py runserver` para ver la página de bienvenida de Django.
- **Verificación del proyecto:** Comprobaremos que los paths de importación de Python reconozcan la carpeta `apps/`.
